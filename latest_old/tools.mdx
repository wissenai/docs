This refers to the tools that are available to be declared

Below is a table of tools & usage notes. Use this as a guide for determining what tools you need to use and understanding tool dependencies. You should still use your own judgement on what tools to utilise.

By standardising tools in this manner, we allow for ourselves to be able to describe tools:
- by action: e.g. `list_*`, `fetch_*`, `extract_*`, `search_*`
- by provider: e.g. `wissen-*`, `quartr-*`, `factset-*`
- by entity: e.g. `*_company_*`, `*_generation_*`

---

| Provider | Action | Tool Name | Returns | Filter By | Usage Notes |
|----------|--------|-----------|---------|-----------|-------------|
| Wissen | check | wissen-check_token_usage | token_usage | — | Use this to monitor your context window. You have a hard limit that you need to be aware of |
| Wissen | create | wissen-create_plan | plan | — | This should be done after you've explored data to detail how you will approach the research |
| Wissen | write | wissen-todo_write | todo | — | Use this to log updates to your work specifically related to the plan created via `wissen-create_plan` |
| Wissen | list | wissen-list_gen_types | gen_types | — | Lists unique generation types available in the database. The `gen_type` values are self-descriptive (e.g., `revenue_analysis`) |
| Wissen | list | wissen-list_generations | generations | `ticker`, `gen_type` | Lists generations available in the database. Always use `ticker` and `gen_type` filters to get specific sets |
| Wissen | fetch | wissen-fetch_generation | generation | `gen_id` | Returns outputs of a specific generation. Use `wissen-list_generations` to get the `gen_id` |
| Wissen | fetch | wissen-fetch_tool_calls | tool_calls | `tool_id`, `gen_id` | Returns tool calling data. Pass `tool_id` for specific tool or `gen_id` for all tools used in a generation |
| Wissen | extract | wissen-extract_quartr_transcript | transcript_excerpts | `url`, `query` | Extracts data from a Quartr transcript URL. Use `query` to specify what information you're looking for |

| Quartr | list | quartr-list_company_events | events | `company_id`, `type_ids`, `start_date`, `end_date` | Get context on events (earnings calls, conferences) for a company |

| Quartr | list | quartr-list_company_transcripts | transcripts | `company_id` | List transcripts (earnings calls, investor conferences) for a company |
| Quartr | list | quartr-list_event_types | event_types | — | Get available event types. Helpful for contextualizing what events reports/slides/transcripts are part of |
| Quartr | list | quartr-list_document_types | document_types | — | Get available document types for better understanding of Quartr documents |

| Fiscal AI | fetch | fiscal_ai-fetch_company_profile | company_profile | `ticker` | Get company profile and compile identifier (`company_key` = `<exchange_symbol>_<ticker>`) for other Fiscal AI tools |

| Fiscal AI | fetch | fiscal_ai-fetch_company_balance_sheet_as_reported | balance_sheet | `company_key`, `period_type` | Get balance sheet as reported. Specify `period_type` |
| Fiscal AI | fetch | fiscal_ai-fetch_company_cash_flow_as_reported | cash_flow | `company_key`, `period_type` | Get cash flows as reported. Specify `period_type` |
| Fiscal AI | fetch | fiscal_ai-fetch_company_ratios | ratios | `company_key`, `ratio_id` | Get company ratios across time. Specify `ratio_id` from `list_ratios` for specific data |
| Fiscal AI | fetch | fiscal_ai-fetch_company_daily_ratios | daily_ratios | `company_key`, `ratio_id`, `start_date`, `end_date` | Get daily ratios. Specify `ratio_id` from `list_ratios` |
| Fiscal AI | fetch | fiscal_ai-fetch_company_shares_outstanding | shares_outstanding | `company_key` | Get company's shares outstanding over time |
| Fiscal AI | fetch | fiscal_ai-fetch_company_segments_and_kpis | segments_kpis | `company_key`, `period_type` | Get company's segments breakdowns and KPIs across time |
| Fiscal AI | fetch | fiscal_ai-fetch_company_adjusted_metrics | adjusted_metrics | `company_key`, `period_type` | Get company's adjusted metrics |
| Fiscal AI | fetch | fiscal_ai-fetch_company_stock_splits | stock_splits | `company_key` | Get company's historical stock splits |
| Fiscal AI | fetch | fiscal_ai-fetch_company_stock_prices | stock_prices | `company_key`, `start_date`, `end_date` | Get company's stock prices. Always specify date range |
| Reducto | extract | reducto-extract_data | extracted_data | `url`, `system_prompt` | Extract data from a PDF URL. Use `system_prompt` to specify what you're looking for. Do not change the `schema`. | Exa | search | exa-search_web | search_results | `query`, `num_results`, `include_domains`, `exclude_domains` | Search for information on the web. Think carefully about credible sources for a hedge fund |
